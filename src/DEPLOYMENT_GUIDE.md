## ‚úÖ Simple 5-Step Deployment Process

### Step 1: Export Your Code from Figma Make
1. In Figma Make, click the **"Export"** or **"Download"** button (usually top-right)
2. Download the ZIP file to your computer
3. Unzip the folder - you'll see all your files

---

### Step 2: Sign Up for Vercel (FREE)
1. Go to: **https://vercel.com**
2. Click **"Sign Up"**
3. Choose **"Continue with GitHub"** (easiest option)
4. Create a free GitHub account if you don't have one
5. Authorize Vercel to access GitHub

---

### Step 3: Prepare Your Code
1. Open the unzipped folder from Step 1
2. Create a new file called **`vercel.json`** in the root folder
3. Copy this into the file:

```json
{
  "buildCommand": "npm install && npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

4. Save the file

---

### Step 4: Upload to GitHub
1. Go to **https://github.com/new**
2. Name your repository: **jioww-global-website**
3. Click **"Create repository"**
4. You'll see instructions - follow the option: **"uploading an existing file"**
5. Drag and drop your entire project folder
6. Click **"Commit changes"**

---

### Step 5: Deploy on Vercel
1. Go back to **https://vercel.com**
2. Click **"Add New Project"**
3. Select your **jioww-global-website** repository
4. Click **"Import"**
5. Keep all default settings
6. Click **"Deploy"**

**Wait 2-3 minutes** - Vercel will build your site.

## Current Limitations in Figma Make

**Social media link previews cannot work in Figma Make** because:
- Figma Make is a client-side React environment
- Meta tags are generated by JavaScript (after page load)
- Social media crawlers need meta tags in the initial HTML (before JavaScript runs)
- Only server-side rendering can fix this

**Google Search Console indexing issues:**
- Figma Make preview URLs are temporary and should NOT be submitted to Google Search Console
- Google cannot properly index client-side React apps without server-side rendering
- Preview environments often have noindex tags or robots.txt restrictions

**Solution**: Deploy to Vercel following this guide!

## ‚úÖ After Deployment - Submit to Google

Once deployed to Vercel with your custom domain:

1. **Go to Google Search Console**: https://search.google.com/search-console
2. **Add your property**: Enter your domain (e.g., jiowwglobal.com)
3. **Verify ownership**: Follow Google's verification steps
4. **Submit sitemap**: 
   - In Search Console, go to "Sitemaps"
   - Enter: `https://www.jiowwglobal.com/sitemap.xml`
   - Click "Submit"
5. **Request indexing**:
   - Go to "URL Inspection"
   - Enter your homepage URL
   - Click "Request Indexing"

**Your site now has:**
- ‚úÖ Proper robots meta tags (`index, follow`)
- ‚úÖ Canonical URLs for each page
- ‚úÖ Structured data (JSON-LD) for rich search results
- ‚úÖ Complete sitemap.xml
- ‚úÖ robots.txt file allowing all search engines
- ‚úÖ SEO-optimized meta tags
- ‚úÖ Open Graph tags for social sharing

**Indexing Timeline:**
- Google typically indexes new sites within 1-4 weeks
- Request indexing in Search Console to speed up the process
- Check "Coverage" report to see indexing status
